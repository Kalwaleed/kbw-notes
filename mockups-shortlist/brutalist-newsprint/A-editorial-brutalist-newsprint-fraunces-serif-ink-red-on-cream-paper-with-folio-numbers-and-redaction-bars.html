The quiet revolution of local-first AI agents.

Six months running an offline-first agent stack on a single M-series laptop — what broke, what didn't, and the surprising case for sovereignty over throughput.

PK
Prince KhaledEDITOR · KBW
28 APR 2026 12 MIN READ

For most of the last decade, the default architecture for "intelligent" software has been cloud-first: a thin client, a fat API, and the unspoken assumption that the network is always available, always cheap, and always trustworthy. That assumption is breaking down — not loudly, but everywhere at once.

Over the last six months I have been running a full agent stack — orchestrator, retrieval, codegen, evaluations — on a single laptop. No API calls. No streaming costs. No 03:00 incident bridge because someone's region went down. The result has been simultaneously underwhelming and revelatory.

1. The setup, briefly

The base layer is a 14B-parameter model running through llama.cpp, plus a 1.3B reranker, plus a small MTEB-tuned embedder for retrieval. Everything fits in 22GB of unified memory with headroom for an IDE, a browser, and one ill-advised Slack window.

Why a smaller model wins

The frontier model is better. The local model is here. For 80% of the agent's day-to-day work — code search, summarization, intent routing, draft writing — those two facts trade places in importance the moment your network coughs.

The frontier is a luxury. Sovereignty is infrastructure.

2. What broke (and what didn't)

  • Embeddings: surprisingly, never. A 384-d Sentence-Transformer model handled 14k chunks without complaint.
  • Tool use: required hand-written grammars. Function-calling parity with cloud models is still 12–18 months out.
  • Long-context recall: degrades hard past 32k tokens. The fix is structural — chunk, summarize, re-rank — not parametric.
  • Energy: a sustained workload draws ~28W. Not free, but cheaper than a single OpenAI bill.

The numbers I care about most:

  1. p95 latency dropped from 2,400ms to 380ms.
  2. Monthly inference cost dropped from $1,840 to $0.
  3. Trust surface shrank to one device under one keychain.

3. A code sketch

The router is roughly 40 lines of TypeScript. Here is the load-bearing piece — note how the local path is default, not fallback:

// routes/agent.ts — local-first dispatcher
import { localModel, cloudModel } from "@kbw/inference";
import { telemetry } from "@kbw/obs";

export async function route(q: Query): Promise<Answer> {
  const budget = q.priority === "interactive" ? 350 : 2000;
  const local  = await withTimeout(localModel(q), budget);
  if (local.ok && local.confidence > 0.82) {
    telemetry.tick("answered.local");
    return local;
  }
  // graceful escalation — only when local is uncertain
  return await cloudModel({ ...q, hint: local.draft });
}

4. What this means for product teams

If you are shipping anything with an LLM in the loop in 2026, you have three options. Build cloud-first and inherit a recurring vendor tax. Build cloud-only and inherit a recurring outage tax. Or build local-first with cloud escalation — and inherit a one-time engineering tax that decays toward zero.

The third option is harder to ship and easier to live with. It is also, I now believe, the only path that compounds.

Closing note

None of this is a manifesto against frontier models. It is a manifesto against treating them as the only available answer. The frontier is a luxury we pay for in dollars and dependence. Sovereignty is infrastructure we build once and own forever.
